# Requisitos del Sistema

## Funcionales
- Registro e inicio de sesión de usuarios.
- Un usuario puede tener varios hogares.
- Gestión de gastos por hogar (luz, agua, gas, internet…).
- Gestión de productos del almacén por hogar.
- Histórico de compras por hogar.

## No funcionales
- Interfaz adaptable a móvil y web.
- Seguridad en autenticación y gestión de datos.
- Documentación clara y actualizada.
- Control de versiones utilizando Git y GitHub.

## Tecnológicos
- Backend: Java con Spring Boot.
- Frontend: React.
- Base de datos: H2 (dev), PostgreSQL (prod).
- Despliegue inicial: Heroku (backend), Vercel/Netlify (frontend).
