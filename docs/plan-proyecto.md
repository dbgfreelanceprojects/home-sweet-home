# Plan del Proyecto

## Fases

1. **Documentación inicial**
2. **Configuración de repositorio y herramientas**
3. **Modelo de datos y backend (API, seguridad, pruebas)**
4. **Frontend y comunicación con backend**
5. **Pruebas integrales y primer despliegue**
6. **Documentación de uso, API y despliegue**
7. **Iterar mejoras y nuevas funciones**

## Herramientas de gestión

- **GitHub Issues** para tareas/incidencias.
- **GitHub Project** (Board Kanban) para organización visual de tareas.
- **Trello** como alternativa si prefieres algo externo.

## Buenas prácticas

- Commits pequeños y descriptivos.
- Uso de ramas por funcionalidad.
- Revisión y actualización constante de la documentación.
